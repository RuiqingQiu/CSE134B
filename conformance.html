Browser compatibility issues

(Note: Tags and elements that are not listed here have no real compatibility issues as far as we tested it.)

1. No issues with the script tag in Firefox, Chrome, or IE. However, Internet Explorer restricted script.html from rendering Javascript until I clicked a message box to "allow" script.html to show Javascript.

2. No issues with the script tag in Firefox, Chrome, or IE. In Internet Explorer, I had to restart the browser for turning off Javascript to take effect.

3. The template tag does not work in Internet Explorer.

4. In form-related tags, Date and time input types is not supported by IE and Firebox but work fine on Chrome. Control bar looks very different in IE vs. Firefox and Chrome. Other than that, forms have no compatibility issues

5. Most of the text-level semantics tags are supported by all three browser. However, bdi tag is not supported by all three browsers.

6. All embedded content elements using audio/video need ActiveX plugin to display in IE.

7. The object element is not supported by Firefox.

8. The param element (used together with object) is not supported by Firefox.

9. The audio/video control bars looks very different on IE.
